# Индивидуальный рабочий репозиторий по курсу C++



Данный репозиторий содержит структуру программного проекта для решения задач на языке программирования C++ для одноименного курса.

Каждый репозиторий является единым решением (solution) и открывается в IDE, поддерживающей современные версии CMake (3.5 и выше), для решения всех задач курса, в том числе добавленных в репозиторий по мере выкладки новых серий задач.



### Структура проекта-репозитория

Репозиторий (каталог верхнего уровня) содержит следующие важные компоненты.

* 📂`/cmake/` — дополнительные модули CMake, необходимые для бесшовной интеграции проекта;
* 📂 `/contrib/` — дополнительные внутренние компоненты, модули и библиотеки, нужные для построения проектов;
  * 📂`/contrib/catch2/` — амальгамизированная версия [библиотеки модульного тестирования Catch2 в форме двух файлов-исходников;
* 📂`/lib/` — дополнительные внешние библиотеки, нужные для построения проектов;
* 📂**`/tasks/`** — подкаталог с задачами для решения;
  * 📂`/multiplication/` — каталог с таргетом одной из задач с названием *multiplication*;
    * `/CMakeLists.txt` — CMake-конфигурация таргета *multiplication*;
    * `/test.cpp` — модульные тесты, проверяющие (тестирующие) корректность реализации таргета *multiplication*;
    * `/multiplication.h` — заголовочный файл, интерфейсная часть модуля *multiplication*, экспортирующая C++-функцию `multiply()`, которую необходимо реализовать для решения одноименной задачи;
    * *`/multiplication.cpp`* — модуль трансляции, реализация модуля *multiplication*, куда необходимо положить реализацию функции `multiply()`; это единственный файл в таргете *multiplication*, который должен (и имеет права) редактировать студент;
  * 📂... — таргет...
  * 📂`/password/` — таргет password...
  * 📂... — таргет...
  * `/CMakeLists.txt` — CMake-конфигурация для каталога с задачами для решения, здесь можно временно исключать из проекта отдельные таргеты — для ускорения сборки или в случае ошибок; 
* **`/CMakeLists.txt`** — CMake-конфигурация верхнего уровня (проекта) — главный конфигурационный файл проекта. Именно он должен открываться (как проект) в IDE (CLion, Qt Creator; либо содержащий его каталог — VS Code).

⚠Важно! Данная структура не должна изменяться или модифицироваться, т.к. это может привести к невозможность сборки проекта либо его последующей проверки. В большинстве случаев, единственные файлы, которые должен редактировать студент в процессе решения задач, это cpp-файлы внутри таргетов подкаталога `/tasks/`, например: *`/multiplication.cpp`*, *`/password.cpp`*, но не файлы `test.cpp`, содержащие модульные тесты для соответствующей задачи.
