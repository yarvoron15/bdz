# LRU Cache

Это алгоритм хранения кэша, который вытесняет из него элементы, к которым обращались дольше всего, чтобы освободить место для новых данных. 

### Цель

Реализовать класс `LRUCache` (файл `lru-cache.h`), который поддерживает операции:

- **clear**
- **get / put / erase / pin / unpin** за $O(1)$ амортизированно;
- операцию **merge** — слияние двух кешей с сохранением порядка элементов внутри каждой из частей.

### Что делают операции?

- **get(key)**: ищет элемент по ключу, если элемент найден — помечает его как «недавно использованный» и возвращает `std::optional<int>` с значением; если не найден — `std::nullopt`;

- **put(key, value)**: если ключ уже в кэше, то обновляет значение и помечает элемент как "недавно использованный", иначе если кэш заполнен — удаляет самый старый элемент и добавляет новый элемент;

- **erase(key)**: удаляет элемент с указанным ключом из кэша, возвращает `true` если элемент был найден и удален, `false` если элемент не найден;

- **pin(key)**: закрепляет элемент, закрепленные элементы не могут быть вытеснены при достижении `capacity`, сохраняет относительный порядок среди закрепленных элементов, возвращает `true` если элемент был найден и успешно закреплен, `false` если не найден или уже был закреплен;

- **unpin(key)**: снимает закрепление с элемента, после снятие элемент может быть вытеснен из кэша, помечает элемент как "недавно использованный" среди обычных элементов, возвращает `true` если элемент был найден и успешно разблокирован, `false` если элемент не найден или уже был разблокирован;

- **merge(other)**: перемещает все элементы из кэша `other` в текущий кэш, сохраняет порядок элементов, удаляет дубликаты: если ключ из `other` уже существует в текущем кэше, элемент из `other` отбрасывается, после слияния выполняется вытеснение (если необходимо)

### Ограничения 

Все операции кроме `merge` должны работать за $O(1)$ амортизированно. Можно использовать только `std::list` и `std::unordered_map`. Ключи будут элементами `std::string`, значения будут элементам `int`.

В ситуации, когда по запрашиваемому ключу значение может отсутствовать, бывает полезно воспользоватся стандартным типом `std::optional<int>`. Данный тип (доступен, начиная с C++17) явно кодирует два состояния — «значение присутствует» и «значение отсутствует» — и тем самым исключает необходимость возвращать «сырые» указатели. Документация: [https://en.cppreference.com/w/cpp/utility/optional](https://en.cppreference.com/w/cpp/utility/optional).
