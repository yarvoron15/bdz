# Транспонирование матрицы

Дан двумерный массив (матрица) размером $m×n$ ($m, n < 1000$). Симметричный ему относительно главной диагонали массив называется транспонированным к данному. Он имеет размеры  $n×m$: строки исходного массива становятся столбцами транспонированного,  столбцы исходного массива становятся строками транспонированного.

Для данного массива постройте транспонированный массив, реализовав функцию `transpose()`.



### Представление матрицы, вход и выход

Для представления матрицы в данном задаче мы будет использовать контейнер `std::vector`, который представляет функционал динамически изменяемого (по размеру) массива. Тип данных `std::vector` является *шаблоном класса*, который параметризуется типом данных — элементами вектора. Если в качестве такого параметра (типа элементов вектора) положить другой вектор (с числовым типом элементов), получится вектор векторов чисел, а это раскладывается в двумерную проекцию, которая подходит для представления двумерного массива или матрицы.

Для более удобной манипуляции с таким типом данных, вводятся псевдонимы для типа строки матрицы и самой матрицы с помощью ключевого слова `using`:

```c++
using Row = std::vector<int>;
using Matrix = std::vector<Row>;
```

На вход разрабатываемой функции `transpose()`  матрица подается с помощью механизма константной ссылки, что исключает необходимости копировать тяжелый объект, когда необходимо только получить некоторые его свойства. Константность обеспечивает контракт, который гарантирует, что функция `transpose()` случайно или специально *не модифицирует переданную матрицу*, поэтому мы можем спокойно передавать туда даже `r-value` -объекты[^rvalue].

В результате транспонирования создается новая матрица, которая возвращается из функции `transpose()` по значению. Такой подход эффективно работает, начиная с C++ стандарта 11, в котором была введена *семантика перемещения* (а сейчас, как правило, к нему присоединяется еще и техника оптимизации *copy elision*).

> При создании новой матрицы позаботьтесь о том, чтобы минимизировать аллокации памяти, т.к. все необходимые размеры уже известны на входе. Это **обязательное условие** для того, чтобы задача была засчитана.

Гарантируется, что на вход подана прямоугольная матрица (то есть длины всех строк равны).

Если на вход будет подана матрица, у которой хотя бы один из размеров равен нулю (т.е. ноль строк или столбцов — что можно определить по длине первой строки), необходимо вернуть объект `Matrix` нулевой длины. 





---


[^rvalue]: Короткоживущие объекты, время жизни которых часто ограничивается выражением, где они появляются. Такие объекты не могут быть привязаны к другим объектам «по соседству» с помощью механизма косвенности — указателем или ссылкой.
