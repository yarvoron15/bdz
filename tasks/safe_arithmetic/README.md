# Безопасная арифметика

## Обзор

Напишите безопасную реализацию основных арифметических операций. В случае успешного вычисления функции возвращают значение `true`, а результат вычисления — с помощью неконстантной ссылки через параметр `result`.  Если результат операции нельзя представить в типе возвращаемого значения, функции возвращают признак ошибки (`false`) и значение `result `не определено.

```cpp
bool safeAdd(int64_t a, int64_t b, int64_t& result);
bool safeSubtract(int64_t a, int64_t b, int64_t& result);
bool safeMultiply(int64_t a, int64_t b, int64_t& result);
bool safeDivide(int64_t a, int64_t b, int64_t& result);
```

## Ограничения
Конечно, в этой задаче (как и в большинстве других) запрещено использовать нестандартные расширения вроде int128_t или builtin-функций. Можно использовать исключительно стандартную библиотеку.



---

*Примечание*. Хотя условия задачи выше и не разрешают использовать 128-битные типы, они все же встречаются в тестах (модуль `test.cpp`), чтобы облегчить проверку корректности реализованных операций. Для компиляции проекта с таким типом может понадобиться 64-битный компилятор.
