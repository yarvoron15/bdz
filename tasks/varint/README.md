# Varint: Кодирование целых чисел переменной длины



## Обзор

В данной задаче вам необходимо выполнить разбор (распарсить) числа, закодированные форматом varint (variable integers). Это очень простой и популярный формат для эффективного представления небольших чисел. Например, число 42 в varint занимает 1 байт вместо, например, 8 байт при хранении в `uint64_t`. Ключевая идея состоит в том, что большинство целых чисел в реальных приложениях невелики. Поэтому мы можем экономить место, используя меньше байтов для меньших чисел.

Varint широко применятся в разных форматах хранения и передачи данных. Например, это Protocol Buffers — популярный формат бинарной сериализации от Google.



## Как работает кодирование Varint

Varint записывается как поток октетов (байт длины ровно 8 бит). **Старший бит (MSB)** каждого байта рассматривается как **бит продолжения**:
- **MSB = 1**: Есть ещё байты
- **MSB = 0**: Это последний байт

Оставшиеся младшие **7 бит** этого байта представляют фактические данные. Байты расположены в порядке **little-endian** (младший байт идет первым).

Можете считать, что в данной задаче байт всегда есть 8 бит.



### Примеры

**Пример 1: Число 150**

```
150 в двоичном виде: 10010110

Разбиваем на 7-битные блоки (справа): 0000001 0010110
Меняем естественный порядок октетов на little-endian: 0010110 0000001
Добавляем биты продолжения: 10010110 00000001
Финальный набор октетов: [0x96, 0x01]
```

**Пример 2: Число 300**

```
300 в двоичном виде: 100101100

Разбиваем на 7-битные блоки: 0000010 0101100
Меняем порядок: 0101100 0000010
Добавляем биты продолжения: 10101100 00000010
Финальный набор октетов: [0xAC, 0x02]
```

**Пример 3: Число 16384**
```
16384 в двоичном виде: 100000000000000

Разбиваем на 7-битные блоки: 0000010 0000000 0000000
Меняем порядок: 0000000 0000000 0000010
Добавляем биты продолжения: 10000000 10000000 00000010
Финальный набор октетов: [0x80, 0x80, 0x02]
```



## Ваша задача

Реализуйте декодер varint, который преобразует последовательности байтов обратно в целые числа. Вам предлагается сделать это в виде одной функции с достаточно сложной сигнатурой:

```cpp
size_t DecodeVarint(const uint8_t* data, size_t size, uint64_t& result);
```

Функция возвращает результат и через возвращаемое значение (число прочитанных байт), и через аргумент `result` (переданный по ссылке[^С]), куда записывается полученное число. Если по какой-то причине не получилось распарсить varint, верните 0, не меняя значение в `result`.

Обратите внимание, что мы принимаем здесь последовательность байт в виде указателя (`data`) и размера (`size`). Вспомните массивы и адресную арифметику! — С этими аргументами можно делать операцию: читать *i*-тый байт последовательности через `data[i]`, если `i < size`. Если же `i >= size`, то чтение приведет к UB.

Так как мы парсим 64-битные числа, varint длиннее 10 октетов можно отвергать. Ваша функция должна корректно обрабатывать некорректные входные данные.

[^С]: см. тж. задачу *associative_operation*. В данном случае ссылка не является константной (перед типом аргумента нет ключевого слова `const`), поэтому через данный аргумент можно напрямую изменять значение объекта — фактического параметра, переданного в вызывающей функции. Тем самым мы как бы «возвращаем» еще одно значение (помимо собственно возвращаемого) из этой функции наружу.
